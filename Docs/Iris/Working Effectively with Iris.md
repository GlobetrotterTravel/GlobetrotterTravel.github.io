# Working Effectively with Iris

This section provides practical guidance for solving common scenarios with Iris.

## Typical entity definition setup

This section will guide you through a typical entity defintion setup, including adding child component definitions and their facet packages.

At a high level, defining an entity and its child component structure consists of:

**Entity**
1. Creating the entity class
1. Creating the entity definition
1. Create a facet package
1. Assign the facet package to the entity

**Fields**
1. Create a field class
1. Create one or more field definitions
1. Create a field definition facet package
1. Assign each field definition to the entity definition
1. Assign the field definition facet package to each field definition

**Hybrid Lists**
1. Create one or more hybrid list definitions
1. Create a hybrid list definition facet package
1. Assign the hybrid list definition facet package to each hybrid list definition
1. Assign each hybrid list definition to the entity definition

**Entitlements**
1. Create one or more entitlement definitions
1. Create an entitlement definition facet package
1. Assign the entitlement definition facet package to each entitlement definition
1. Assign each entitlement definition to the entity definition

**Assets**
1. Create one or more asset definitions
1. Create an asset definition facet package
1. Assign the asset definition facet package to each asset definition
1. Assign each asset definition to the entity definition
